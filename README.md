# Earthquake Classifier

A Computational Science project that uses machine learning to distinguish **earthquake** signals from **noise** in raw seismometer recordings, using the [Stanford Earthquake Dataset (STEAD)](https://www.kaggle.com/datasets/isevilla/stanford-earthquake-dataset-stead/data).

The project has two parts:

1. **Classifier** — binary classification of a 60-second seismogram as *earthquake* vs *noise*, comparing a classical signal-processing baseline against a small 2D CNN trained on log-power spectrograms.
2. **Picker** — a regression CNN that, given an earthquake waveform, predicts the sample index of the P-wave arrival.

## Results

### Earthquake / noise classifier (test set)

| Model | F1 | Accuracy | Precision | Recall | AUC-ROC |
|---|---|---|---|---|---|
| Classical baseline (peak-amplitude threshold, 6.527σ) | 0.9043 | 0.9047 | 0.8661 | 0.9335 | 0.9576 |
| **SpectrogramCNN** (10×10 log-power spectrogram) | **0.9977** | **0.9977** | **0.9978** | **0.9971** | **0.9999** |

### P-wave arrival picker (test set)

| Model | MAE |
|---|---|
| PickerCNN (1D CNN on raw waveform) | 33.7 samples ≈ 0.337 s @ 100 Hz |

## Dataset

Data comes from [STEAD](https://www.kaggle.com/datasets/isevilla/stanford-earthquake-dataset-stead/data):

- `chunk1` — noise traces
- `chunk2` — local earthquake traces (`trace_category == "earthquake_local"`)

Each chunk is an `(.hdf5, .csv)` pair: the HDF5 file holds 3-channel (E/N/Z), 6000-sample (60 s @ 100 Hz) waveforms, and the CSV holds per-trace metadata (magnitude, distance, depth, P/S arrival samples, SNR, etc.).

After processing (`src/data/process_stead.py`), only the **Z (vertical) channel** is kept, z-score normalised per trace. Traces with NaNs/Infs, zero variance, or clipped amplitudes are dropped.

| | Total | Earthquakes | Noise |
|---|---|---|---|
| Train | 256,691 | 116,084 | 140,607 |
| Val   | 64,158  | 29,007  | 35,151  |
| Test  | 106,921 | 48,333  | 58,588  |
| **All** | **427,770** | **193,424** | **234,346** |

- Earthquake traces are split **by event** (`source_id`), so all traces from the same earthquake stay in one split.
- Noise traces are split **per trace**.
- Split ratios: 60% train / 15% val / 25% test.
- 193,424 earthquake traces come from **98,746 unique events**, magnitude range ≈ [-0.1, 6.2], epicentral distance range ≈ [0.04, 334] km.

## Data pipeline

All commands are run from the project root.

1. **Download STEAD chunk1 (noise) and chunk2 (earthquakes)** from Kaggle and unzip them so each lands in its own folder:

   ```
   raw_data/chunk1/chunk1.hdf5, chunk1/chunk1.csv
   raw_data/chunk2/chunk2.hdf5, chunk2/chunk2.csv
   ```

2. **Build the waveform + metadata arrays:**

   ```bash
   python -m src.data.build_data
   ```

   Produces `data/waveforms.npy` and `data/metadata.csv`.

3. **Build spectrogram patches** (default 10×10 grid; spectrograms computed with `scipy.signal.spectrogram`, `fs=100`, `nperseg=256`, `noverlap=200`, converted to dB and patch-averaged):

   ```bash
   python src/data/build_spectrograms.py --n 10
   ```

   Produces `data/spectrograms/spectrograms_10x10.npy`.

4. **Compute dataset statistics** (counts, split breakdown, magnitude/distance/depth/SNR summaries, global spectrogram dB range for plotting):

   ```bash
   python -m src.data.statistics
   ```

   Produces `data/statistics.json`.

## Training & evaluation

### Classifier

```bash
# Classical peak-amplitude baseline
python -c "from src.classifier.classical import find_threshold, evaluate; evaluate(find_threshold())"

# CNN on spectrograms
python -m src.classifier.train      # saves checkpoints/classifier_best.pt
python -c "from src.classifier.evaluate import compute_metrics, error_analysis; \
compute_metrics('checkpoints/classifier_best.pt'); error_analysis('checkpoints/classifier_best.pt')"
```

`SpectrogramCNN` (`src/classifier/model.py`) is a small 2D CNN over the 10×10 log-power spectrogram: three conv blocks (1→32→64→128 channels) with batch norm, ReLU, max-pooling and adaptive average pooling, followed by a `512 → 256 → 1` linear head producing a single logit. Trained with `BCEWithLogitsLoss`, Adam, `ReduceLROnPlateau`, and early stopping on validation F1.

### Picker

```bash
python -m src.picker.train          # saves checkpoints/picker_best.pt
python -c "from src.picker.evaluate import evaluate; evaluate('checkpoints/picker_best.pt')"
```

`PickerCNN` (`src/picker/model.py`) is a 1D CNN over the raw 6000-sample waveform: three conv blocks (1→16→32→64 channels) with ReLU and 4× max-pooling, flattened (not pooled, so positional information is preserved) into a single linear unit predicting the P-arrival sample index. Trained with `L1Loss` (MAE), Adam, `OneCycleLR`, and AMP on CUDA. `PickerDataset` loads only the ~116K earthquake traces with a labelled P-arrival into RAM for fast training.

## Notebooks

- **`src/classifier/notebooks/classifier_classical.ipynb`** — explores the `max|x|` / kurtosis feature distributions, sweeps thresholds on the train set, and evaluates the chosen threshold on the test set.
- **`src/classifier/notebooks/classifier_cnn.ipynb`** — trains `SpectrogramCNN`, plots training curves, and runs the full test-set evaluation and FN/FP error analysis.
- **`src/picker/notebooks/picker.ipynb`** — trains and evaluates `PickerCNN`, with residual histograms and example predictions.
- **`src/notebooks/test.ipynb`** — exploratory visualisation of raw waveforms and spectrograms for earthquake vs. noise examples.

## Notes

- `raw_data/`, `data/`, and `checkpoints/` are gitignored — the data pipeline and training scripts regenerate them from the downloaded STEAD chunks.
- All spectrogram plots share a global dB color scale (`p2`/`p98` percentiles from `data/statistics.json`) so colors are comparable across plots.
